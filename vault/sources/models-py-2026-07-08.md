# MLPG Data Models (Pydantic)

`src/models.py` (49 lines) defines 6 Pydantic models that form the validation contract between OpenRouter LLM responses and the MLPG application.

## Model Hierarchy

```
TutorResponse (root)
└── PercorsoStudio
    ├── Metadati (difficolta_impostata, objective_apprendimento)
    └── Modulo[] (id, titolo_modulo, spiegazione, esercizio_pratico)

FeedbackValutazione (standalone, response format)
└── esito: corretta | parziale | sbagliata

RiepilogoFinale (standalone, summary format)
└── punti_di_forza, punti_da_migliorare, diario_di_bordo, saluto_conclusivo
```

## Key Design Decisions

### Strict Validation
All models use `model_config = {"extra": "forbid"}` to reject any unexpected fields from the LLM. This catches hallucinated keys early and prevents silent data corruption.

### Optional Fields in Feedback
`FeedbackValutazione` uses `Optional[List[str]]` for punti_di_forza, punti_migliorabili, and errors_comprensione — these may not always be present in LLM responses. Only `commento_costruttivo` and `suggerimento_miglioramento` are required.

### Required Lists in Summary
`RiepilogoFinale` requires both `punti_di_forza: List[str]` and `punti_da_migliorare: List[str]`. This enforces that the LLM must always provide strengths AND weaknesses — no empty summaries.

### Module Max Length
`Modulo.spiegazione` is capped at 2500 characters via `Field(..., max_length=2500)`. This prevents the LLM from generating excessively long explanations that would slow down rendering or cost too many tokens.
